Hallucination Patterns

  • Tests that check elements exist but never interact with them
  • expect(x).toBe(x) — tautologies that can never fail
  • Phantom selectors targeting non-existent DOM elements
  • Impressive suites with no meaningful assertions

5. Guardrail Framework

✅ AUTOMATE FREELY

  • Test data generation
  • Code formatting
  • Reports
  • Log parsing
  • Environment setup

⚠️ GATE (Human Reviews)

  • Test case generation
  • Defect triage
  • Spec review
  • Code suggestions
  • Coverage analysis

🚫 PROHIBIT (Human Only)

  • Security decisions
  • Compliance sign-off
  • Risk acceptance
  • GDPR data handling
  • Release approval

5 Guardrail Principles

  1. Never commit AI code without human review
  2. Every test needs a meaningful business logic assertion
  3. Every positive test → at least one negative/boundary test
  4. Every test links to a spec requirement (traceability)
  5. Manual exploration first — understand before generating
If you can't explain what an AI-generated test validates, don't commit it.

8. Data Privacy — Quick Rules

✅ Can Share with AI

  • Open-source patterns
  • Generic logic
  • Synthetic test data
  • Anonymized descriptions

🚫 Cannot Share

  • Client code (without permission)
  • Real customer data / PII
  • NDA-covered requirements
  • Security vulnerabilities

Rule: When in doubt — anonymize or synthesize. Never assume it's safe.
